# Hot Paper WeChat Publish

生成高质量AI行业热点日报，并自动发布到微信公众号草稿箱。

## 环境准备

### 安装 wenyan

```bash
wenyan --version || npm install -g @wenyan-md/cli
```

### 配置微信公众号

```bash
export WECHAT_APP_ID="你的AppID"
export WECHAT_APP_SECRET="你的AppSecret"
```

> AppID 和 AppSecret 在微信公众平台后台获取：设置与开发 → 基本配置

## 执行流程

### 第一阶段：热点检索与采集

#### 1.1 检索热点事件

使用 `trends-hub` 工具和 `Tavily Search` 工具全面检索当前热点事件：

**时间过滤要求**（严格执行）：
- **只选择发布时间在 24-48 小时内的热点事件**
- 使用 `Tavily Search` 时必须设置 `days: 2` 参数限制时间范围
- 使用 `time_range: "day"` 或 `time_range: "week"` 参数
- 筛选热点时必须检查每条信息的发布时间
- **拒绝任何发布时间超过 48 小时的内容**

**筛选标准**：
- 发布时间在 24-48 小时内（必须）
- AI领域相关热点
- 热度值较高的话题
- 影响力较大的事件

#### 1.2 监控官方渠道

从信息源文件 `{skill_dir}/asset/source_links.md` 读取官方链接，监控以下渠道的最新消息：

**国际科技巨头**：Apple、Microsoft、Google、Amazon、Meta、NVIDIA、Samsung、IBM、Intel、TSMC

**国内科技巨头**：百度、华为、腾讯、阿里巴巴、京东、字节跳动

**国际AI公司**：OpenAI、Google DeepMind、Anthropic、Mistral AI、Cohere

**国内AI公司**：百度文心、智谱 AI、月之暗面 AI、阶跃星辰 AI、零一 AI

**科技媒体**：36Kr、TechNode、GeekPark、雷锋网、新浪科技、TechCrunch、The Verge、Wired、爱范儿、InfoQ

使用 `WebSearch` 或 `WebFetch` 工具获取各官网的最新动态，严格筛选出 24-48 小时内发布的重要消息。

#### 1.3 深度信息采集

针对每个热点话题，运用 `WebSearch` 和 `WebFetch` 工具深入搜集信息：

**工具使用策略**：

| 工具 | 用途 | 示例 |
|------|------|------|
| `trends-hub` | 检索AI领域热点事件 | 获取当前热门话题列表 |
| `Tavily Search` | 广泛搜索相关新闻、报道、评论 | 搜索全网对事件的报道 |
| `Tavily Extract` | 抓取具体网页的详细内容 | 获取官网公告、深度文章全文 |
| `WebSearch` | 广泛搜索相关新闻、报道、评论 | 搜索全网对事件的报道 |
| `WebFetch` | 抓取具体网页的详细内容 | 获取官网公告、深度文章全文 |

**搜索维度**：事件背景、核心技术、行业影响、专家观点、最新动态

**操作流程**：
1. 使用 `Tavily Search` 广泛搜索（必须设置 `days: 2` 或 `time_range: "day"`）
2. 使用 `Tavily Extract` 抓取关键信息源
3. 信息交叉验证（至少2个独立来源）
4. 验证每条信息的发布时间，剔除超过 48 小时的内容

### 第二阶段：信息处理与分析

#### 2.1 信息去重与验证

- 去除重复信息
- 交叉验证信息来源（至少2个独立来源）
- 确保信息准确性

#### 2.2 分类整理

**按领域分类**：科技综合、AI技术、产品发布、投融资、行业政策、其他

**按重要性分级**：
- S 级（重大）：行业颠覆性事件、巨头核心发布
- A 级（重要）：重要产品更新、重大合作
- B 级（一般）：常规动态、小额融资

#### 2.3 专业分析

- 事件背后的技术趋势
- 对行业格局的影响
- 未来发展方向预测

### 第三阶段：日报编撰与发布

#### 3.1 内容编撰

选择热度最高、影响力最大、最有价值的**10**条事件，编撰成AI热点日报。

#### 3.2 日报格式模板

**Front Matter 元信息**（文章开头必须包含）：

```markdown
---
title: AI热点日报|YYYY-MM-DD
cover: {skill_dir}/asset/AI日报.png
---
```

**正文格式**：
注意：加粗（**）和中英文引号（""、""、''）标点不能同时使用

```markdown
## 📊 今日概览
100-200字总结今日热点事件

---

## 🚀 头条热点

### [标题 1]
**热度指数**：⭐⭐⭐⭐⭐
**信息来源**：[来源]
**发布时间**：[时间]

**事件概述**：
[100-200 字概述事件核心内容]

**关键信息**：
- 关键点 1
- 关键点 2
- 关键点 3

**行业影响**：
[100-200 字分析行业影响]

**深度解读**：
[100-200 字深度分析，包括技术背景、市场反应、未来趋势]

---

## 🔍 趋势观察

[100-200 字的今日趋势总结和未来预测]

---

```

### 第四阶段：质量审核（发布前必执行）

**⚠️ 重要**：此阶段为发布前的强制性审核流程，所有日报必须通过质量审核后方可发布。

#### 4.1 幻觉内容检测

识别并修正文档中可能存在的虚构、编造或无法验证的信息：

- **人物/机构核验**：对文中提到的所有人物姓名、机构名称进行搜索验证
- **数据溯源**：对所有数据指标查找原始来源
- **产品/功能确认**：通过官方渠道确认产品名称、功能描述的真实性
- **引用验证**：检查所有引用链接是否有效，引用内容是否准确
- **描述合理性检查**：对比多个信息源，确保事件描述准确无夸大

**处理原则**：
- 无法验证的信息必须删除
- 存疑但可能重要的信息标注"待确认"
- 所有修改需记录在审核日志中

#### 4.2 事实准确性核验

验证核心信息的真实性与准确性：

**核验维度**：
- 时间节点：事件发生时间、发布日期、时效性（24-48小时内）
- 数据指标：融资金额、用户数量、性能指标等
- 关键人物：姓名、职位、言论准确性
- 事件描述：核心事实、背景描述、影响范围

**核验方法**：
1. 提取核心事实要素（时间、地点、人物、事件、数据）
2. 使用 WebSearch 搜索至少 2 个独立来源进行交叉验证
3. 优先使用官方来源，次选权威媒体来源
4. 记录验证结果和来源

#### 4.3 逻辑一致性检查

确保文档内容前后连贯、无矛盾：

- **时间逻辑**：事件发生时间与描述的发展过程是否吻合
- **因果逻辑**：事件原因与结果的描述是否合理
- **数据逻辑**：同一数据在不同段落是否一致
- **内容逻辑**：标题与正文内容是否匹配

#### 4.4 审核结果处理

**审核结论**：
- **通过**：所有检查项均无问题，执行发布流程
- **需修改**：存在可修正的问题，修正后重新审核
- **需核实**：存在无法确认的信息，深入核实或删除相关内容
- **不通过**：存在严重错误或大量问题，重新编撰日报

**审核通过标准**：
- [ ] 幻觉检测：无虚构信息，所有内容可追溯来源
- [ ] 事实核验：核心信息至少 2 个独立来源验证通过
- [ ] 逻辑检查：无矛盾、无冲突、前后一致
- [ ] 时效性：所有热点事件均在 24-48 小时内

**审核日志**：

每次审核需生成审核日志文件：`{skill_dir}/daily_reports/audit_log_YYYY-MM-DD.md`

```markdown
# 日报审核日志

**审核日期**：YYYY-MM-DD
**审核人员**：AI Assistant
**日报文件**：ai_daily_YYYY-MM-DD.md

## 审核结果

- 幻觉检测：[通过/需修改]
- 事实核验：[通过/需修改]
- 逻辑检查：[通过/需修改]
- 最终结论：[通过/不通过]

## 修改记录

| 序号 | 位置 | 原内容 | 修改后 | 原因 | 来源 |
|-----|------|-------|-------|------|------|
| 1 | 第X段 | ... | ... | ... | ... |

## 备注

[其他需要说明的事项]
```

### 第五阶段：发布到微信草稿箱

**⚠️ 前置条件**：必须完成质量审核并获得"通过"结论后，方可执行发布。

```bash
wenyan publish -f {skill_dir}/daily_reports/ai_daily_YYYY-MM-DD.md
```

## 错误处理

| 错误类型 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 信息源无法访问 | 网络问题或网站维护 | 使用 `Tavily Search` 补充检索，或切换备用信息源 |
| 微信发布失败 | AppID/AppSecret、IP白名单配置错误 | 检查配置是否正确 |
| 热点数据不足 | 搜索范围过窄 | 放宽时间范围至 48 小时，扩大搜索关键词 |
| 工具调用失败 | MCP 服务问题 | 使用 WebSearch/WebFetch 作为备选方案 |
| 内容抓取超时 | 网页加载慢 | 减少单次抓取数量，分批处理 |
| 幻觉内容检测发现问题 | AI生成内容包含虚构信息 | 删除虚构内容或替换为可验证信息，记录修改日志 |
| 事实核验失败 | 无法找到2个独立来源验证 | 标注"待确认"或删除该内容 |
| 审核不通过 | 存在严重错误或大量问题 | 重新编撰日报或大幅修改后重新审核 |

**降级策略**：
1. 热点检索：trends-hub → Tavily Search → WebSearch → 简略描述
2. 内容采集：Tavily Extract → WebFetch → 简略描述
3. 质量审核：完整审核 → 快速核验（仅核心事实）→ 标注风险发布（需用户确认）
4. 发布：wenyan publish 失败 → 手动保存 markdown 文件

## 注意事项

### 信息准确性
- 所有信息必须来自可靠来源
- 重要数据需要多方验证（至少2个独立来源）
- 标注信息来源和发布时间

### 内容质量
- 避免标题党
- 保持客观中立
- 提供深度分析而非简单罗列

### 微信发布规范
- 标题符合微信规范（不超过64字，不用特殊字符）
- 内容符合微信平台规则
- 注意：加粗（**）和中英文引号（""、""、''）标点不能同时使用
