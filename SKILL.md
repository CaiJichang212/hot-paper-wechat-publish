# Hot Paper WeChat Publish

本技能用于生成高质量的AI行业热点日报，并自动发布到微信公众号草稿箱。

## 功能概述

- **热点检索**：全面检索AI领域的当前热点事件
- **信息采集**：深入搜集每个热点话题的详细信息
- **系统整理**：系统化整理与专业化分析信息
- **日报生成**：生成结构清晰、内容详实的热点日报
- **质量审核**：全面审核日报内容，确保准确性和可靠性
- **自动发布**：自动发布到微信公众号草稿箱

## 环境准备

### 1.1 安装 wenyan

wenyan 是微信公众号发布的命令行工具。

```bash
# 验证是否已经安装，避免重复安装
wenyan --version

# 使用 npm 全局安装
npm install -g @wenyan-md/cli

```

### 1.2 配置微信公众号

```bash
# 设置 AppID 和 AppSecret
export WECHAT_APP_ID="你的AppID"
export WECHAT_APP_SECRET="你的AppSecret"
```

> **提示**：AppID 和 AppSecret 需要在微信公众平台后台获取。登录微信公众平台 → 设置与开发 → 基本配置。


## 执行流程

### 第一阶段：热点检索与采集

#### 1.1 检索热点事件

使用 `trends-hub` 工具和 `Tavily Search` 工具全面检索当前热点事件：

**⚠️ 时间过滤要求（严格执行）**：

- **只选择发布时间在 24-48 小时内的热点事件**
- 使用 `Tavily Search` 时必须设置 `days: 2` 参数限制时间范围
- 使用 `time_range: "day"` 或 `time_range: "week"` 参数
- 在筛选热点时必须检查每条信息的发布时间
- **拒绝任何发布时间超过 48 小时的内容**

**筛选标准**：

- 发布时间在 24-48 小时内（必须）
- AI领域相关热点
- 热度值较高的话题
- 影响力较大的事件

#### 1.2 监控官方渠道

读取并监控以下官方渠道的最新消息：

**信息源文件**：`{skill_dir}/asset/source_links.md`

**监控范围**：

**国际科技巨头**：Apple、Microsoft、Google、Amazon、Meta、NVIDIA、Samsung、IBM、Intel、TSMC

**国内科技巨头**：百度、华为、腾讯、阿里巴巴、京东、字节跳动

**国际AI公司**：OpenAI、Google DeepMind、Anthropic、Mistral AI、Cohere

**国内AI公司**：百度文心、智谱 AI、月之暗面 AI、阶跃星辰 AI、零一 AI

**科技媒体**：36Kr、TechNode、GeekPark、雷锋网、新浪科技、TechCrunch、The Verge、Wired、爱范儿、InfoQ

**监控方法**：

1. 从信息源文件（`{skill_dir}/asset/source_links.md`）读取官方链接
2. 使用 `WebSearch` 或 `WebFetch` 工具获取各官网的最新动态
3. **严格筛选出 24-48 小时内发布的重要消息**
4. 在采集信息时，优先使用带时间过滤参数的搜索工具

#### 1.3 深度信息采集

针对筛选出的每个热点话题，运用 `WebSearch` 和 `WebFetch` 工具深入搜集信息：

**工具使用策略**：

| 工具          | 用途             | 示例            |
| ----------- | -------------- | ------------- |
| `WebSearch` | 广泛搜索相关新闻、报道、评论 | 搜索全网对事件的报道    |
| `WebFetch`  | 抓取具体网页的详细内容    | 获取官网公告、深度文章全文 |

**搜索维度**：

- **事件背景**：事件起因、发展历程
- **核心技术**：技术原理、创新点
- **行业影响**：对行业的影响、市场反应
- **专家观点**：行业专家评价、分析师观点
- **最新动态**：最新进展、后续计划

**操作流程**：

1. 使用 `Tavily Search` 广泛搜索（必须设置 `days: 2` 或 `time_range: "day"`）
2. 使用 `Tavily Extract` 抓取关键信息源
3. 信息交叉验证
4. **验证每条信息的发布时间，剔除超过 48 小时的内容**

#### 2.1 信息去重与验证

- 投融资

- 事件背后的技术趋势

- 未来发展方向预测

### 第四阶段：质量审核（发布前必执行）

**⚠️ 重要提示**：此阶段为发布前的强制性审核流程，所有日报必须通过质量审核后方可发布。未经审核的日报禁止发布。

#### 4.1 幻觉内容检测

**检测目标**：识别并修正文档中可能存在的虚构、编造或无法验证的信息。

**检测范围**：

| 检测项目 | 检测方法 | 处理方式 |
|---------|---------|---------|
| 虚构人物/机构 | 交叉验证来源 | 删除或替换为可验证信息 |
| 编造的数据指标 | 查找原始数据源 | 标注"数据待确认"或删除 |
| 不存在的产品/功能 | 官方渠道核实 | 删除虚构内容 |
| 错误的引用来源 | 验证链接有效性 | 修正或删除引用 |
| 夸大的事件描述 | 多源对比 | 修正为准确描述 |

**检测流程**：

1. **人物/机构核验**：对文中提到的所有人物姓名、机构名称进行搜索验证
2. **数据溯源**：对所有数据指标（如融资金额、用户数量、性能指标等）查找原始来源
3. **产品/功能确认**：通过官方渠道确认产品名称、功能描述的真实性
4. **引用验证**：检查所有引用链接是否有效，引用内容是否准确
5. **描述合理性检查**：对比多个信息源，确保事件描述准确无夸大

**处理原则**：
- 无法验证的信息必须删除
- 存疑但可能重要的信息标注"待确认"
- 所有修改需记录在审核日志中

#### 4.2 事实准确性核验

**核验重点**：验证核心信息的真实性与准确性。

**核验维度**：

**时间节点核验**：
- 事件发生时间是否准确
- 发布日期是否正确
- 时间线是否合理
- 时效性是否符合要求（24-48小时内）

**数据指标核验**：
- 融资金额、估值等财务数据
- 用户数量、增长率等运营数据
- 性能指标、技术参数
- 市场份额、排名等

**关键人物核验**：
- 人物姓名、职位是否正确
- 人物言论是否准确引用
- 人物与事件的关联是否属实

**事件描述核验**：
- 事件核心事实是否准确
- 事件背景描述是否正确
- 事件影响范围是否准确

**核验方法**：

对于每条热点新闻：
1. 提取核心事实要素（时间、地点、人物、事件、数据）
2. 使用 WebSearch 搜索至少 2 个独立来源进行交叉验证
3. 优先使用官方来源（官网、官方账号、官方公告）
4. 次选权威媒体来源（知名科技媒体）
5. 记录验证结果和来源


**核验记录模板**：

| 核验项目 | 文中内容 | 验证来源1 | 验证来源2 | 核验结果 |
|---------|---------|----------|----------|---------|
| 时间 | YYYY-MM-DD | 来源URL | 来源URL | ✓/✗ |
| 数据 | XX亿 | 来源URL | 来源URL | ✓/✗ |
| 人物 | 张三，CEO | 来源URL | 来源URL | ✓/✗ |

#### 4.3 逻辑一致性检查

**检查目标**：确保文档内容前后连贯、无矛盾。

**检查项目**：

**时间逻辑一致性**：
- [ ] 事件发生时间与描述的发展过程是否吻合
- [ ] 多个相关事件的时间顺序是否合理
- [ ] "近日"、"昨日"等时间表述是否准确

**因果逻辑一致性**：
- [ ] 事件原因与结果的描述是否合理
- [ ] 影响分析的逻辑链条是否完整
- [ ] 预测与现状的关联是否合理

**数据逻辑一致性**：
- [ ] 同一数据在不同段落是否一致
- [ ] 数据之间的关系是否合理（如增长率计算）
- [ ] 数据单位是否统一

**内容逻辑一致性**：
- [ ] 标题与正文内容是否匹配
- [ ] 概述与详细描述是否一致
- [ ] 分类标签与内容是否匹配

**检查方法**：

1. **通读检查**：完整阅读全文，标记可疑的矛盾点
2. **要素提取**：提取所有时间、数据、人物要素
3. **交叉比对**：对比相同要素在不同位置的出现
4. **逻辑推演**：验证因果关系和推理链条

#### 4.4 审核结果处理

**审核结论分类**：

| 结论 | 条件 | 处理方式 |
|-----|------|---------|
| 通过 | 所有检查项均无问题 | 执行发布流程 |
| 需修改 | 存在可修正的问题 | 修正后重新审核 |
| 需核实 | 存在无法确认的信息 | 深入核实或删除相关内容 |
| 不通过 | 存在严重错误或大量问题 | 重新编撰日报 |

**修改记录要求**：

每次修改必须记录：
- 修改位置（段落/行号）
- 原始内容
- 修改后内容
- 修改原因
- 验证来源

**审核通过标准**：

- [ ] 幻觉检测：无虚构信息，所有内容可追溯来源
- [ ] 事实核验：核心信息至少 2 个独立来源验证通过
- [ ] 逻辑检查：无矛盾、无冲突、前后一致
- [ ] 时效性：所有热点事件均在 24-48 小时内

**审核日志**：

每次审核需生成审核日志文件：`{skill_dir}/daily_reports/audit_log_YYYY-MM-DD.md`

## 修改记录

| 序号 | 位置 | 原内容 | 修改后 | 原因 | 来源 |

| 1 | 第X段 | ... | ... | ... | ... |

## 核验记录

## 错误处理

### 常见错误及解决方案

| 错误类型    | 可能原因                      | 解决方案                             |
| ------- | ------------------------- | -------------------------------- |
| 信息源无法访问 | 网络问题或网站维护                 | 使用 `Tavily Search` 补充检索，或切换备用信息源 |
| 微信发布失败  | AppID/AppSecret,IP白名单配置错误 | 检查配置是否正确                         |
| 热点数据不足  | 搜索范围过窄                    | 放宽时间范围至 48 小时，扩大搜索关键词            |
| 工具调用失败  | MCP 服务问题                  | 使用 WebSearch/WebFetch 作为备选方案     |
| 内容抓取超时  | 网页加载慢                     | 减少单次抓取数量，分批处理                    |
| 幻觉内容检测发现问题 | AI生成内容包含虚构信息 | 删除虚构内容或替换为可验证信息，记录修改日志 |
| 事实核验失败 | 无法找到2个独立来源验证 | 标注"待确认"或删除该内容，继续核验其他内容 |
| 逻辑矛盾发现 | 时间线、数据或描述前后不一致 | 修正矛盾内容，重新进行逻辑检查 |
| 审核不通过 | 存在严重错误或大量问题 | 重新编撰日报或大幅修改后重新审核 |

### 降级策略

1. **热点检索降级**：trends-hub → Tavily Search → WebSearch
2. **内容采集降级**：Tavily Extract → WebFetch → 简略描述
3. **质量审核降级**：完整审核 → 快速核验（仅核心事实） → 标注风险发布（需用户确认）
4. **发布降级**：wenyan publish 失败 → 手动保存 markdown 文件

## 注意事项

### 信息准确性

- 所有信息必须来自可靠来源
- 重要数据需要多方验证（至少2个独立来源）
- 标注信息来源和发布时间

### 时效性要求

- **严格优先选择 24 小时内的热点**
- **绝对不超过 48 小时**
- 标注事件发生时间
- **在编撰日报前，必须再次检查每条热点的发布时间**
- **如果某条热点超过 48 小时，必须替换为更新的热点**

### 质量审核要求

- **质量审核为发布前的强制性流程，不可跳过**
- 所有内容必须可追溯来源，无法验证的信息必须删除
- 审核发现的问题必须修正后方可发布
- 审核日志必须完整记录所有修改和核验过程
- 存在严重错误的日报必须重新编撰

### 内容质量

- 避免标题党
- 保持客观中立
- 提供深度分析而非简单罗列

### 微信发布规范

- 标题符合微信规范（不超过64字，不用特殊字符）
- 内容符合微信平台规则
- 注意：加粗（**）和中英文引号（“”、""、''）标点不能同时使用
